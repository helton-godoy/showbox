**Diretório `tests`:**

Este diretório é dedicado exclusivamente ao armazenamento dos scripts responsáveis pelos testes automatizados, incluindo aqueles integrados aos pipelines de CI/CD, além de testes que serão executados manualmente pelo desenvolvedor para análise em tempo real dos resultados do desenvolvimento. Os testes automatizados devem cobrir cenários de unidade, integração e regressão, garantindo a qualidade e estabilidade do código. Preferencialmente, os testes devem utilizar os scripts de exemplo disponíveis em `@/exemples/` e `@/exemples/demos` quando a intenção for validação visual, verificação de sucesso na execução dos elementos empregados nos scripts ou demonstração de funcionalidades. Recomenda-se organizar os testes em subdiretórios por tipo (ex.: `unit/`, `integration/`, `manual/`), e incluir arquivos de configuração para facilitar a execução local e em ambientes de CI/CD. Todos os testes devem ser documentados com comentários claros e, quando aplicável, incluir asserções detalhadas para facilitar a depuração.
