**Diretório `src`:**

Este diretório é dedicado exclusivamente ao armazenamento dos artefatos do ambiente de desenvolvimento. Ele será criado dentro de um contêiner Docker, com um volume do contêiner mapeado para montagem no diretório `@/src/code/`. Esse diretório será responsável por concentrar todo o código-fonte do showbox, ao mesmo tempo em que permite acesso direto ao código-fonte tanto pelo Docker quanto pelo host do desenvolvedor. Essa estratégia visa possibilitar a construção de um ambiente de desenvolvimento reproduzível de forma padronizada, rápida e fácil. O desenvolvedor precisará apenas fazer o clone do projeto e executar o script `@/tools/start-docker-dev.sh` para iniciar o desenvolvimento do `showbox`. O script terá a função de identificar o sistema operacional em uso, verificar a presença do Docker e, caso não esteja instalado, sugerir a instalação. Imediatamente após essa mensagem, o script solicitará confirmação do usuário para permitir o início da instalação do Docker. Se confirmado, o script executará a instalação do Docker conforme as orientações presentes na documentação oficial para cada sistema operacional específico. Uma vez que tudo estiver pronto, o script inciará o conteiner Docker contendo todas as dependências de desenvolvimento do projeto já disponíveis. O ambiente de desenvolvimento dentro do contêiner permitirá que as execuções dos scripts de exemplo (`@/examples`), quando realizadas de dentro do contêiner Docker, sejam exibidas no host principal para visualização e validação pelo desenvolvedor.
